\section{NorthMacedonia\_IA samples}
\label{si:samples}

The 14 unique \texttt{NorthMacedonia\_IA} individuals used here were
published in \citet{Lazaridis2022}. We introduce this cluster only as an
additional 1-source candidate column under the same present-day qpAdm setup
documented by \citet{Olalde2023}. As noted in the main text, this is not a
later or peripheral source but one drawn from the same immediate authorial and
evidentiary context as the published substrate set. We do not infer from that
inclusion that it is a literal historical ancestor of any modern population.
The exact sample IDs assigned to this and the other Olalde-defined source
clusters are recorded in \nolinkurl{data/cluster_definitions.tsv}.

\section{Strict target panel}
\label{si:targets}

The printed local 1-source rows in Olalde Data~S2 Table~8 are
\texttt{Croatian}, \texttt{Serbian}, \texttt{Romanian},
\texttt{Bulgarian}, \texttt{Albanian}, \texttt{Greek\_Macedonia},
\texttt{Greek\_Peloponnese}, \texttt{Greek\_Cyclades}, \texttt{Cretan}, and
\texttt{Greek\_Dodecanese}. This comment does not relabel those printed
rows to nearby public proxies. In particular, it does not treat printed
\texttt{Serbian} as identical to public \texttt{Serbian\_Serb}, and it does
not collapse the printed regional Greek labels onto broader public labels such
as \texttt{Greek} or \texttt{Cypriot}. The strict public-HO panel reported here
therefore consists of the exact public labels \texttt{Croatian},
\texttt{Serbian\_Serb}, \texttt{Romanian}, \texttt{Bulgarian},
\texttt{Albanian}, \texttt{Greek}, and \texttt{Cypriot}. The retained target
panel is shown in Table~\ref{tab:target-panel}.

\section{Interpretive caution}

A 1-source qpAdm pass under this right-set is a non-rejection statement, not
a proof of continuity. The contribution of this comment is methodological
rather than historical: within this strict public-label restatement, it shows
that the published local-substrate set is not exhaustive, because an omitted
Iron Age Balkan candidate changes the pass/fail pattern materially when
inserted into the same first-stage test.
